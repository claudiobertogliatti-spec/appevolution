**Key Technical Concepts**
-   **Asynchronous Task Queues (Celery & Redis):** Using a distributed task queue to handle long-running, unreliable processes (like video generation) without blocking the main application.
-   **Self-Managed Services:** A creative solution () to run and monitor background services (Celery worker and beat) as Python subprocesses, bypassing the limitations of a read-only  file in the managed environment.
-   **Architectural Abstraction (Provider Pattern):** The  interface decouples the core business logic of analysis generation from the specific AI implementation (Claude), allowing for future flexibility (e.g., swapping in NotebookLM).
-   **Event-Driven Architecture (Webhooks):** Shifting from a fragile polling mechanism to a robust, event-driven model for payment verification using Stripe webhooks.
-   **Database Views (MongoDB Aggregation):** A pragmatic approach to address architectural debt by creating a unified, read-only view () of fragmented data without performing a risky data migration.
